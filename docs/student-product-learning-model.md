# 学生端产品学习模型

本文记录学生端首页、学习、AI、测评、我的五个入口的产品逻辑。它不是 UI 细节稿，而是用于约束后续原型、路由和内容组织方式的心智模型。

## 核心判断

学生端不是多邻国式闯关学习，也不是单纯视频库。它更准确的定位是：

```text
以实验视频为入口、以课本目录为地图、以 AI 和测评形成闭环的实验学习系统。
```

学生的核心心智是：

```text
我先看到一个实验现象，想知道它是什么；
再回到课本目录理解它属于哪一章；
不懂就问 AI；
最后用测评确认自己会不会。
```

## 五个入口的职责

```text
首页 = 实验视频发现流
学习 = 课本实验地图
AI = 随身实验助教
测评 = 薄弱点诊断器
我的 = 学习记录本
```

### 首页

首页应当是实验视频页，而不是复杂功能聚合页。它负责让学生快速看到化学实验现象，例如颜色变化、沉淀、气体、置换反应、焰色反应等。

由于系统素材主要是横屏实验视频，首页不应采用竖屏 Shorts 形态。更合理的方向是 Bilibili / YouTube 首页式横屏视频流：

```text
16:9 实验视频封面或静音预览
实验标题
课本目录路径
元素 / 试剂 / 现象标签
进入详情 / 收藏 / 问 AI
```

交互上可以采用自动预览：

```text
用户上下滚动
  -> 某个视频卡片进入主视区
  -> 该视频静音自动播放
  -> 离开主视区后暂停
  -> 下一条进入主视区后播放
```

首页的目标不是让学生沉迷刷视频，而是让学生通过实验现象进入课本实验点位。

### 学习

学习页是分类目录模型，不是引导学习模型。

它负责回答：

```text
这个实验在课本体系里的什么位置？
我想按章节、元素、族、分区找实验，应该从哪里进入？
```

目录模型应保持中立，类似文件系统或教材目录：

```text
第 13 章 卤族元素
  -> 二、卤素的氧化性
    -> 氯水与 KBr 置换实验
```

目录不负责判断学生应该学什么，只负责说明内容在哪里。

### AI

AI 不是独立聊天玩具，而是带上下文的实验助教。

AI 应尽量携带当前上下文：

```text
实验标题
课本目录路径
元素 / 试剂 / 现象标签
反应方程式
实验目的
实验原理
学生当前测评薄弱点
```

学生从视频详情、目录点位、测评结果进入 AI 时，AI 应知道学生正在问哪一个实验或知识点。

### 测评

测评不是目录结构的一部分，而是推荐和诊断来源。

测评系统可以根据学生表现生成薄弱点：

```text
第 13 章得分低
卤素氧化性相关题目错误率高
氯水与 KBr 置换实验相关概念不稳定
```

再把学生带回对应目录节点或实验点位：

```text
推荐复习：二、卤素的氧化性
推荐观看：氯水与 KBr 置换实验
推荐练习：卤素氧化性相关题
```

推荐系统只引用目录节点和点位，不应重建一套学习目录。

### 我的

我的页负责沉淀学习轨迹：

```text
看过的视频
收藏的实验
最近学习
薄弱章节
测评记录
老师反馈
```

它不是主要学习入口，但应帮助学生回到未完成、已收藏或需要补弱的内容。

## 原型逻辑流

最终学生端应支持三类入口路径：

```text
路径一：发现型
首页刷到实验视频
  -> 进入视频详情
  -> 查看实验目的、原理、用品、方程式
  -> 问 AI 或进入相关目录

路径二：任务型
学习页按课本 / 元素 / 目录找实验
  -> 进入目录节点
  -> 打开实验点位
  -> 看视频和实验解释

路径三：诊断型
完成测评
  -> 系统识别薄弱章节或点位
  -> 推荐相关目录节点 / 实验视频
  -> 学生复习后再次测评
```

三条路径最终都汇入同一个核心内容：

```text
实验点位详情 = 视频 + 现象 + 方程式 + 原理 + AI + 测评关联
```

## 横屏视频详情

首页视频流只做预览和发现。点开视频后可以进入实验学习播放器。

横屏时推荐布局：

```text
┌──────────────────────────────┬───────────────┐
│                              │ 实验攻略       │
│                              │ 实验目的       │
│        横屏实验视频           │ 实验原理       │
│                              │ 实验用品       │
│                              │ 做实验 / 问 AI │
└──────────────────────────────┴───────────────┘
```

竖屏时推荐布局：

```text
上：16:9 视频
下：实验目的 / 原理 / 用品 / 方程式 / AI / 测评动作
```

右侧栏或下方信息区应聚焦三要素，不要变成完整教材页面：

```text
实验目的
实验原理 / 方程式
实验用品 / 安全提醒
```

## 关于“娱乐化”的准确解释

这里的风险不是产品一定会娱乐化，也不是实验视频本身有问题。

风险来自短视频形态的外部误读：

```text
如果首页只有自动播放视频、点赞、收藏、无限下滑，
但没有课本路径、实验目的、知识标签、进入详情、测评回流，
用户和老师会把它理解成娱乐短视频流。
```

因此更准确的说法是：

```text
不是娱乐化风险，而是短视频交互形态被误读为娱乐流的风险。
```

避免误读的产品约束：

- 每条首页视频必须露出课本目录路径。
- 每条视频必须能进入实验点位详情。
- 视频标签应优先使用元素、试剂、现象、章节，而不是泛娱乐话题。
- 首页自动播放应默认静音，作为实验预览，不作为完整学习承载。
- 详情页必须回到实验目的、原理、用品、方程式和 AI 解释。
- 测评推荐应把学生带回目录节点或实验点位，而不是生成新的视频流分类。

## 与旧五级原型的区别

旧原型把内容画成固定五级：

```text
周期表
  -> 族
    -> 性质分类
      -> 实验验证类型
        -> 具体实验
```

这适合讲故事，但不适合作为当前系统的真实信息架构。当前系统的目录深度不是固定五级，而是目录树：

```text
chapter
  -> directory
    -> directory
      -> point
```

正确边界是：

```text
页面承载上下文
目录承载层级
点位承载内容
搜索承载检索
测评承载推荐
AI 承载解释
```

## 产品原则

- 用视频吸引学生进入。
- 用目录告诉学生内容在哪里。
- 用 AI 帮学生解释为什么。
- 用测评判断学生会不会。
- 用推荐把学生带回薄弱点。
- 不把分类目录伪装成固定学习路径。
- 不让首页视频流脱离课本实验上下文。
